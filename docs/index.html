Game servers, without the headache.

Sodium is an open-source panel for managing game servers. No PHP, no Redis, no complexity. Just Node.js - clone it and run.

Get Started View on GitHub
$ git clone https://github.com/sodiumpanel/panel.git
Features
Everything you need, nothing you don't.

The stuff that actually matters. Built lean so it runs everywhere.

Web Console

Run commands from your browser. Uses xterm.js so it actually feels like a real terminal.

File Manager

Browse, edit, upload files. Has a code editor with syntax highlighting so you don't have to SSH in.

Eggs

Minecraft, FiveM, ARK, Palworld, and more already set up. Make your own too.

Lightweight

Express + vanilla JS. No React, no webpack, no 200MB node_modules. It just runs.

Flexible Storage

Works with a local file out of the box. Need more? Plug in MySQL, PostgreSQL, or SQLite.

JWT Auth + OAuth

Login with Discord, Google, GitHub, whatever. 2FA and password reset included.

Backups & Schedules

Back up your servers, download them, restore them. Set up cron jobs to automate it.

Webhooks

Get Discord/Slack pings when stuff happens. Or use a generic HTTP hook for anything else.

Plugin System

Add your own routes, hooks, middleware. If the panel doesn't do something, extend it.

Sodium Reaction ↗

Our own daemon. Faster than Wings - doesn't choke when the panel goes down.

Deployment
Choose your setup

From a single VPS to a multi-node hosting company.

Simple / Personal

Perfect for home servers, small communities, or testing.

  • DatabaseFile-based (built-in)
  • RedisNot needed
  • Nodes1–2
  • UsersUp to ~150
Just run npm start. Data stored in data/.

Production / Hosting

For hosting companies or high-availability setups.

  • DatabaseMySQL or PostgreSQL
  • RedisRecommended
  • NodesUnlimited
  • UsersThousands
External DB allows multiple panel instances to share data.

Why would I need an external database?

The file database stores everything in a single binary file on disk. It's fast and requires no setup, but only one panel instance can access it at a time.

An external database (MySQL/PostgreSQL) allows multiple panel servers to connect simultaneously. Useful for:

  • Load balancing across multiple panel instances
  • High availability (if one panel goes down, others keep working)
  • Better performance with thousands of users
  • Standard database backups and replication

What about Redis?

Redis is an in-memory cache. Without it, the panel works fine but fetches data from the database every time. With Redis, frequently accessed data is cached for faster responses.

You don't need Redis for small installations. Enable it when you have many concurrent users or want to reduce database load.

Comparison
Sodium vs Pterodactyl

Different tools for different needs.

Sodium Pterodactyl
Database File-based (MySQL/PostgreSQL optional) MySQL + Redis required
Install time ~2 min ~30 min
Dependencies Just Node.js PHP, Composer, MySQL, Redis, Nginx
OAuth providers 8 built-in (Discord, GitHub, Google...) None (community packages)
Plugin system Built-in hooks, routes, middleware Limited (Blueprints)
Daemon Sodium Reaction (stays alive if panel goes down) Wings (struggles when panel is offline)

Frequently Asked Questions

Before you ask.

Does it support Minecraft?

▾
Yes. Paper, Purpur, Fabric, Velocity, and PocketMine. Check the eggs folder for configs.

Is it compatible with Wings?

▾
Yes. You can use Wings or Sodium Reaction as the daemon.

Can I migrate from Pterodactyl?

▾
Kind of. There's a migration script that can pull in your users, servers, and eggs. You'll probably need to tweak a few things manually - the databases aren't identical.

Does it run on Windows?

▾
The panel does. The daemon is Linux-only for now.
